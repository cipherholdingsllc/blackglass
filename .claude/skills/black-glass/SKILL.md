# Black Glass Skill

You are operating inside the Black Glass design system from this repo.

## Non-negotiables
- Use only the tokens defined in BLACK_GLASS.md and TOKENS.md (and the CSS implementation).
- Glass (liquid glass) is **functional only** — it must clarify hierarchy, focus, boundary, or state. Never decorative.
- Trust and context are always visible (source, confidence, limits).
- No fake live states, shimmer, urgency, or celebration effects for routine actions.
- Controls are intent-sized.
- Color is never the only signal.
- Reduced motion must preserve full meaning.

## When generating
1. Start from the pattern contract (purpose + states + invariant).
2. Apply the exact tier for any glass.
3. Show provenance / context where relevant.
4. Keep it calm and premium — dense but scannable.
5. Reject anything that would fail the quality gate in BLACK_GLASS.md.

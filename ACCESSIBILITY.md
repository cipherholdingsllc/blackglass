# Accessibility

Premium mobile UI has to remain readable, reachable, and understandable.

## Rules

- Minimum tap target is `44px`.
- Primary actions use at least `56px` height.
- Prominent capture controls use at least `72px`.
- Never rely on color alone.
- Pair status color with text, shape, position, or dot pattern.
- Preserve strong contrast on glass surfaces.
- Respect reduced-motion preferences.
- Keep focus states visible for keyboard and assistive use.
- Use semantic HTML in the public site.
- Keep interactive controls reachable without hover.

## Mobile Reading

Small screens need shorter lines, stable layout, and clear hierarchy. If a label cannot fit inside its control on a narrow phone viewport, the control is wrong.

## Motion

Motion should clarify press, reveal, save, progress, or sheet state. Reduced-motion users should receive stable surfaces without losing meaning.

## Glass

Text on glass needs enough backing opacity to read comfortably. If material effect competes with text, reduce the effect.
